# AI 漫剧/电影/视频生成 Agent - 系统架构

## 概述

基于 Toonflow-app 增强的一站式 AI 视频生成系统。用户只需输入一句话，Agent 自动完成从头到尾的全流程创作，
过程中通过交互式确认节点让用户参与风格和细节决策。

## 核心组件设计

### 1. OneSentenceAgent (新增)

一句话入口 Agent，是用户与系统之间的第一交互层。

**职责：**
- 接收用户一句话创意（如："一个程序员穿越到古代用代码改变世界"）
- 调用 LLM 自动扩写为完整故事大纲
- 引导用户确认创作风格（画风、色调、叙事节奏）
- 自动判断输出类型：漫剧 / 短视频 / 电影短片
- 将确认后的参数传递给下游 Agent

**交互流程：**
```
用户: "一个程序员穿越到古代用代码改变世界"

OneSentenceAgent:
 ├→ 扩写故事大纲（3-5个场景）
 ├→ 询问风格偏好：画风？色调？节奏？
 │  用户: "水墨风，轻松搞笑节奏"
 ├→ 询问输出类型：漫剧 / 视频 / 电影
 │  用户: "漫剧"
 └→ 传递给 ScriptAgent 开始生成
```

### 2. ScriptAgent (增强)

现有决策层 → 执行层 → 监督层三层架构，新增：

**新增能力：**
- `one_shot_script`: 直接从一句话/故事大纲生成完整剧本
- `interactive_confirm`: 在每个关键节点暂停，让用户确认
- `style_injection`: 用户指定的风格注入到剧本中

### 3. ProductionAgent (增强)

现有生产管线，新增：

**新增能力：**
- `manga_mode`: 漫剧生产模式
  - 漫画分格布局
  - 动态效果（运镜、缩放、平移）
  - 特效字幕
  - 转场动画
- `interactive_storyboard`: 分镜交互确认
- `multi_style_output`: 同一内容多风格输出

### 4. 交互式确认节点系统 (新增)

在关键流程节点插入用户确认点：

```
生成流程:
OneSentence → 故事大纲 → [风格确认] → 剧本生成 → [角色确认]
→ 分镜生成 → [分镜预览确认] → 资产生成 → 视频生成 → [成品预览]
                                                          ↓
                                                    [满意度确认]
                                                          ↓
                                                    导出/重新生成
```

## 数据流

```
用户输入 "一句话"
     │
     ▼
[OneSentenceAgent] 扩写 + 风格引导
     │
     ▼
[交互确认] 用户确认风格/角色/输出类型
     │
     ▼
[ScriptAgent] 生成剧本
     │
     ▼
[ProductionAgent] 生成分镜 → 资产 → 视频
     │
     ▼
[预览/导出] 最终成品
```
